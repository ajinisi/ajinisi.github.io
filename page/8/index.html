打开MySQL的文档第十五章 [Chapter 15 The InnoDB Storage Engine] (https://dev.mysql.com/doc/refman/8.0/en/innodb-storage-engine.html) ，好吧全英文，接下来是重点内容的翻译

InnoDB

InnoDB是一种同时兼顾高可靠性和高性能的通用存储引擎。在MySQL 8.0中，InnoDB是默认的MySQL存储引擎。其DML操作遵循 ACID 模型， 具有提交，回滚和崩溃恢复功能的事务，以保护用户数据。

索引

Git的使用之多人合作一

很早就开始使用git了，但是都是一个人写，提交，推送，没有与人合作过。现在在一台电脑上模拟下该过程，即我们要在一台电脑配置多个账号，而这是通过SSH keys实现的。

SSH keys

首先是SSH，而不是SHH，“空眼”患者要注意了。什么是SSH，github有两种推送代码的方式，要么用户名和密码，要么使用SSH keys。为了省去每次访问都要输入用户名和密码的麻烦，采用SSH公钥，密钥对即SSH keys来验证你是否为合法用户。

怎么使用SSH keys

在你的电脑生成了一个唯一的ssh公钥和私钥，公钥放到github上面，当你推送的时候，git就会匹配你的私钥是否跟github上面的公钥是配对的，正确就认为你是合法的，允许推送。现在来确认下你是否已经拥有SSH keys。 默认情况下，用户的 SSH 密钥存储在其 ~/.ssh 目录下。 Windows10系统下，用户的 SSH 密钥存储在其 C:\Users\yourusername.ssh 目录下。

id_rsa就是我们的私钥，id_rsa.pub就是我们的公钥。

  • 如果你没有keys（或者根本没有 .ssh 目录），又或者你不想使用已有的keys，则你可以创建一个新的SSH keys。在一台电脑上使用两个不同的账号，就是采取的此方法建立两个不同的keys。详情见下。

  • 如果你有keys并且想用它连接到Github，则在ssh-agent和Github账户中添加它

新建SSH keys

  1. 使用你的邮箱作为标签，创建一个新的SSH key
  2. 输入文件名，用于保存刚刚生成的 SSH key 代码。不输入将使用 /c/Users/you/.ssh/id_rsa 作为默认文件（如果你是在 ~/.ssh/ 路径下操作的话）。
  3. 输入密码，这个密码会在你提交项目时使用，如果为空的话推送项目时则不用输入。这个设置是防止同一台电脑上的别人私自往你的项目里推送内容。
  4. 因为默认只读取id_rsa，为了让SSH识别新产生的私钥，需将其添加的ssh-agent中，这里要确保你的ssh-agent正在运行
  5. 创建并配置 config 文件，还没有结束，如果现在使用 $ ssh -T git@github.com 只会“欢迎”一个用户。我们还需要添加一个配置文件。

Web之RESTful风格的实现

现在RESTful风格大火。。。结合我自己的项目谈谈我对RESTful风格的使用。

（资源）表现层状态转移

  • 资源：RESTful架构是基于资源的架构（ROA, Resource-Oriented Architecture），在ROA中，处理的对象都是资源。任何需要被引用的对象，都是资源。资源表现为某个具体的URI

  • 表现/表述：所谓表述，指的是资源的某种形式的表示，这个表示不一定是所有信息，可以只是关注的部分信息。并且，同一个资源，可以有多个表述。例如，对于一个景点，可以用jpeg照片来表示，也可以用包含位置、介绍等信息的json或xml格式来分别表示

  • 状态：状态其实应该分为应用状态和资源状态。
    应用状态由客户端保存维护，例如会话状态等。客户端通过REST API返回的表述，以及表述中的URI，进行客户端应用状态的转移。
    但REST更强调的是资源状态。资源状态存储在服务器端，客户端通过REST API，指定请求方法、资源路径和资源表述（可以包含应用状态），对资源的状态进行增删查改。通过增删查改，引起资源状态的改变，称为状态转移

客户端通过REST API对服务器端的资源进行增删查改，引起资源的状态转移。而这种转移是体现在表述上的，所以称为表述性状态转移。

还不够直观？看url就知道要什么，看http method就知道干什么，看http status code就知道结果如何

举个例子，账号的登入登出注册注销，如果按照传统的写法，/login，/logout，/register，/cancell这四个，如果按照RESTful的写法，登入登出对应的服务端资源应该是session，注册注销对应的资源是user，

  • GET /session # 获取会话信息
  • POST /session # 创建新的会话（登入）
  • PUT /session # 更新会话信息

  • DELETE /session # 销毁当前会话（登出）

  • GET /user/:id # 获取id用户的信息

  • POST /user # 创建新的用户（注册）
  • PUT /user/:id # 更新id用户的信息
  • DELETE /user/:id # 删除id用户（注销）
